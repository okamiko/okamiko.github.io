Subscription Splitter

— MongoDB Express.js Node.js

"Can I join your group?"

×
Subscription Splitter | Used MongoDB Express.js Node.js

"Can I join your group?"

Subscription Splitter is a web application that aims to serve as a platform so anyone can find people to split the cost of a variety of services such as Netflix, Chegg, HBO, or Hulu. Users must create an acccount before they are able to make a post. The app allows users to leave comments to let people contact each other. Check out the website here.

  • What we accomplished:
    • Used MongoDB to set up models for Schemas and associated the data using either embedded or referenced methods for one to one relationship, one to many, or many to many relationships. An example would be the comment Schema which defined the model of a comment to have the author and description of the comment. Later, we had to associate comments by exporting it as a model to the group Schema and displaying it on the show route. Finally, after authentication was done, comments were then associated with users when the user signed up and then signed in to add comments
    • Used environment variables so that production and development databases are unique. Database served on mlab and site hosted on Heroku became the production data, while locally we had a separate MongoDB database for testing purposes on AWS Cloud9
    • Used RESTful Routes and mapped HTTP routes and CRUD using Express.js
    • Used Node.js and packages such as body-parser, connect-flash, ejs, express, method-override, mongoose, and passport
    • Used Bootstrap styling methods and its grid system
    • For our project management method, we followed the SCRUM methodology and had weekly standups and burndown charts. Familiarized ourselves with Trello and organized our Sprints there

Classroom Manager

— C++

"Who overslept"

×
Classroom Manager | C++

"First program in C++"

Consider you have an embedded application to be deployed on a set of card readers for a classroom seats management and attendance system. An institute deploys this application on a set of 8 classrooms, with each classroom having 256 available seats. Seats are pre-assigned to students, where by the same seat will be occupied by the same student in all classrooms; i.e. no two students can have the same seat number ever. The seat number of the student will be hardwired on his card. Each student will stamp upon entering and upon leaving a specific class. A student cannot exist in two classes at any point in time. The whole idea is to be able to know how many students are there in any particular class, inquire about a specific student location in any class at any point of time, and to be able also to inquire on the availability of a specific student at any point in time. The maximum number of students that can go to any class is 256, and students are numbered from 0 to 255. Check out the repo here.

"Housing ain't cheap here, mate."

×
Shroommates | Made with Unreal Engine 4.17.2 (User Test Coordinator, Generalist Programmer)

"Mushroom!"

Save your mushroom family from the humans in Shroommates, a light-hearted 3D platforming adventure! Solve puzzles through clever use of your environment and get everyone back home safely. You're a little shroom in a big world - squish down for a powerful jump, parachute away with your mushroom cap, and climb vines to get to vantage points around the house. Beware of the human chef bent on putting delicious shrooms on his pizza, and a deadly Shroomba that attacks fungi on sight! Shroommates is full of funny interactions to discover, catchy music, and a fully voiced cast of colorful characters to meet! Click this to watch the game trailer. Click this to download and play.

  • Implemented game behaviors as needed in various areas, such as:
    • Camera pans and cutscenes to better guide the player to an objective which were accomplished using trigger box interactions.
    • Credits – created an animated sequence for the credits in UE4 which slides the image’s y position from 0 to 3402 in twenty-one seconds.
    • Created a day night cycle which lasted five and a half minutes for one cycle. Also used this to help make the trailer for the first four seconds. Last but not least, added in clouds, stars, and a sun – which were all tweakable values to either enhance or diminish the effects of certain elements in the cycle.
    • Environmental lighting, level design(lamps and mirrors with crisp reflection), and shadows/reflections in the world. Also, a drop shadow on the character mushroom to aid the player in platforming. Added LMIV to efficiently handle build time of light.
    • Game objects: ability to turn on/off lights, bounce on beds or other spongy objects, and open doors which checked if the player has the key or not which then rotated the door ninety degrees.
    • Menus - included correct textures and sounds for the buttons that were either being unselected, hovered, or selected. Implemented resolution settings and support for both keyboard, mouse, and controller. Also included tutorial page, back to title, and continue buttons. Lastly, included switching images based on the event of how you have died (Roomba attacked you or timer ran out).
    • Shop system - controller support for correct textures and a check of twenty coins meant player can buy CD.

Unreal Engine SFX Interactions

— heavy, Wwise, and Pure Data

"Shroommates addons"

×
"Unreal Engine SFX Interactions" | heavy, Wwise, and Pure Data

"Realistic Crickets"

As this project was an extension of the 170 series (Shroommates the game, where the three of us had planned to implement this audio work), I decided it was better placed in a demo world where the other conflicting sounds of the world don’t overwhelm and drown out the sounds. All the sounds were implemented in Pure Data, imported into heavy, then configured with Wwise into Unreal Engine 4.

Locations all in regards to spawn point:
  • Jump sounds based on jump height (charge jump in Shroommates, emulated with bounce pad): Straight ahead of the player, the grey rectangular platform
  • Boombox that holds out music is directly to the left and down the hill - changes tempo during the night
  • Lights that give off electric sounds are scattered across the floating islands - run into them to interact, becomes more sporadic and harsh before they break and disappear
  • Cricket noises come out at night, retreat by day
  • Demo available on request
